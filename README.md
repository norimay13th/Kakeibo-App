# 家計簿アプリ

レシート撮影 or 手入力だけで、Googleスプレッドシート(`¥¥¥`)の「ログ」タブに記録が追記される個人用PWA。
Mac・iPhoneどちらもブラウザ(ホーム画面に追加)で同じアプリとして使えます。サーバーは不要、完全無料です。

## できること

- カメラでレシートを撮影 → 端末に写真を保存せず、その場でOCR（Tesseract.js、無料・オフライン）が店名・金額を仮入力
- 手入力でもすぐ追加できる
- 保存するとGoogleスプレッドシートの「ログ」タブに1行追記される
- 既存の月次タブ・資産推移ダッシュボードには一切手を加えない

## セットアップ手順

### 1. スプレッドシート側の準備

1. 家計簿として使っている `¥¥¥` スプレッドシートを開く
2. 新しいタブを作成し、タブ名を `ログ` にする
3. 1行目にヘッダーを入力する：

   | A | B | C | D | E | F | G | H |
   |---|---|---|---|---|---|---|---|
   | 記録日時 | 対象日付 | 収支種別 | カテゴリ | 内容・店名 | 金額 | 入力方法 | メモ |

4. スプレッドシートのURL `https://docs.google.com/spreadsheets/d/【ここ】/edit` の【ここ】部分（スプレッドシートID）を控えておく

### 2. Google Cloud側の準備（Sheets APIを使うための設定）

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセスし、新規プロジェクトを作成（名前は何でもよい、例: `kakeibo-app`）
2. 左メニュー「APIとサービス」→「ライブラリ」から **Google Sheets API** を検索して有効化
3. 「APIとサービス」→「OAuth同意画面」を開き、User Type は **外部** を選択して作成
   - アプリ名: 家計簿アプリ（任意）
   - ユーザーサポートメール・デベロッパー連絡先: 自分のメールアドレス
   - スコープの追加は不要（後述のクライアントID発行時に指定するため）
   - テストユーザーに自分のGoogleアカウント（`matsui@anrealage.com`など、実際にスプレッドシートを開くアカウント）を追加
   - 公開ステータスは「テスト」のままでOK（自分だけが使うため審査は不要）
4. 「APIとサービス」→「認証情報」→「認証情報を作成」→「OAuthクライアントID」
   - アプリケーションの種類: **ウェブアプリケーション**
   - 承認済みのJavaScript生成元に、後述のホスティング先URLを追加
     - GitHub Pagesの場合: `https://<GitHubユーザー名>.github.io`
     - ローカル確認用に `http://localhost:8000` も追加しておくと動作確認がしやすい
   - 作成後に表示される「クライアントID」（`〜.apps.googleusercontent.com`という文字列）を控える

### 3. アプリ側の設定

`config.js` を開き、以下を書き換える：

```js
const CONFIG = {
  GOOGLE_CLIENT_ID: "（手順2で取得したクライアントID）",
  SPREADSHEET_ID: "（手順1で取得したスプレッドシートID）",
  SHEET_NAME: "ログ",
};
```

> `GOOGLE_CLIENT_ID` と `SPREADSHEET_ID` はどちらも秘密情報ではありません（クライアントIDはブラウザアプリ向けに公開される前提の識別子で、書き込みには利用者本人のGoogleログインが別途必要です）。そのためこのファイルはリポジトリに含めて公開して問題ありません。

### 4. ローカルで動作確認

```bash
cd ~/kakeibo-app
python3 -m http.server 8000
```

Macのブラウザで `http://localhost:8000` を開き、サインイン→手入力→保存→スプレッドシートに反映されるか確認する。

### 5. デプロイ（GitHub Pages）

1. GitHubに新規リポジトリを作成（公開リポジトリでOK。コードに秘密情報は含まれない）
2. このフォルダの中身をpush
3. リポジトリの Settings → Pages → Branch を `main` / `/(root)` に設定して保存
4. 数分後、`https://<GitHubユーザー名>.github.io/<リポジトリ名>/` でアクセスできるようになる
5. その本番URLをGoogle CloudのOAuthクライアント設定（手順2）の「承認済みのJavaScript生成元」に追加する

### 6. iPhoneのホーム画面に追加

1. iPhoneのSafariでデプロイ先URLを開く
2. 共有ボタン →「ホーム画面に追加」
3. ホーム画面のアイコンから起動すると、フルスクリーンのアプリのように使える
