# 家計簿アプリ

Googleスプレッドシート(`¥¥¥`)に手入力で記録している「支出」「収入」「資産」「ローン」「借金」「固定費」を読み取り、
月次の家計簿と年次の資産推移を可視化する個人用PWA。**読み取り専用**で、スプレッドシートへの書き込みは一切行わない。
Mac・iPhoneどちらもブラウザ(ホーム画面に追加)で同じアプリとして使える。サーバー不要、完全無料。

スプレッドシートは「リンクを知っている人は誰でも閲覧可」の共有設定になっているため、**サインインは不要**。制限付きAPIキーだけでスプレッドシートを読み取る。

## できること

- **支出ダッシュボード**（`index.html`）: 月を選んで、今月の収入・支出・貯金額・純資産額・資産額・借金+ローン額、
  カテゴリー別支出の円グラフ、先月比較表を確認できる
- **資産推移**（`trend.html`）: 年を選んで、純資産額と、収入・支出・貯金額・借金+ローン額・資産額の年間推移（1〜12月）、
  現金/株式の保有割合を確認できる
- カテゴリーの固定リストは「ルール」シートから毎回自動で読み取るので、カテゴリーを増やしてもコード変更は不要
- 資産・ローン・借金のようにスナップショットで記録する値は、その月に未記入でも直前に記録した月の値を自動で引き継ぐ

## スプレッドシートの構成

`¥¥¥` スプレッドシートに以下の7タブを用意し、月見出し（例: `2026年7月`）で区切って手入力で記録していく：

| タブ名 | 列構成 |
|---|---|
| 支出 | カテゴリ／内容／金額（月→日→データ行） |
| 収入 | 給与形態／名称／金額 |
| 資産 | カテゴリー(現金/株式)／名称／金額 |
| ローン | 項目名／開始日／総額／分割回数／済回数／残回数／残高／引き落とし |
| 借金 | 項目名／残債務／返済額／返済日 |
| 固定費 | 名称／金額 |
| ルール | `・カテゴリ名` の行でカテゴリー固定リストを定義（`→`から始まる行は説明文） |

## セットアップ手順

### 1. Google Cloud側の準備（Sheets APIを使うための設定）

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセスし、新規プロジェクトを作成（名前は何でもよい、例: `kakeibo-app`）
2. 左メニュー「APIとサービス」→「ライブラリ」から **Google Sheets API** を検索して有効化
3. 「APIとサービス」→「認証情報」→「認証情報を作成」→「APIキー」を選択し、キーを発行する
4. 発行されたキーの「キーを制限」を開き、以下の2つを設定する（誰でもこのキーで自分のSheets APIクォータを使えてしまうのを防ぐため必須）
   - **API制限**: 「キーを制限」→ **Google Sheets API** のみにチェック
   - **アプリケーションの制限**: **ウェブサイト**（HTTPリファラー）を選択し、以下を追加
     - ローカル確認用: `http://localhost:8000/*`
     - デプロイ後の本番URL（例: `https://<GitHubユーザー名>.github.io/*`）も、デプロイできたタイミングで追加する
5. 「キーを表示」で `AIza...` から始まる文字列を控える

### 2. アプリ側の設定

`config.js` を開き、`GOOGLE_API_KEY` を書き換える：

```js
const CONFIG = {
  GOOGLE_API_KEY: "（手順1で取得したAPIキー）",
  SPREADSHEET_ID: "1UsPdLJs_s1miIROKsYfRf20GTI-xCV_WLRYHMLYq8rc",
  DASHBOARD_SHEETS: { ... },
};
```

> `GOOGLE_API_KEY` と `SPREADSHEET_ID` はどちらも秘密情報ではありません。APIキーは「Sheets APIのみ・特定ドメインのみ」に制限してあり、スプレッドシート自体も「リンクを知っている人は誰でも閲覧可」の設定なので、このアプリが読めるのはもともと誰でも見られる範囲のデータだけです。
>
> スプレッドシート側でシート名を変更した場合は、`DASHBOARD_SHEETS` の該当する値も合わせて書き換える。

### 3. ローカルで動作確認

```bash
cd ~/kakeibo-app
python3 -m http.server 8000
```

Macのブラウザで `http://localhost:8000` を開き、サインインなしで統計カードやグラフが表示されるか確認する。

### 4. デプロイ（GitHub Pages）

1. GitHubに新規リポジトリを作成（公開リポジトリでOK。コードに秘密情報は含まれない）
2. このフォルダの中身をpush
3. リポジトリの Settings → Pages → Branch を `main` / `/(root)` に設定して保存
4. 数分後、`https://<GitHubユーザー名>.github.io/<リポジトリ名>/` でアクセスできるようになる
5. その本番URLを、手順1で作ったAPIキーの「アプリケーションの制限」に追加する

### 5. iPhoneのホーム画面に追加

1. iPhoneのSafariでデプロイ先URLを開く
2. 共有ボタン →「ホーム画面に追加」
3. ホーム画面のアイコンから起動すると、フルスクリーンのアプリのように使える
